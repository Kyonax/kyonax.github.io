* WHAT IS KYO-WEB-ONLINE?
Personal portfolio for *[[https://github.com/Kyonax][Cristian D. Moreno - Kyonax]]* — a multilingual (EN / ES) cyberpunk-themed single-page web app showcasing experience, active projects, and contact channels.

The codebase migrated from native Web Components + Webpack to *Vue 3 (=<script setup>= + Composition API) + Vite* in May 2026. Architectural conventions mirror the sibling [[https://github.com/Kyonax/reckit][reckit]] project — same SCSS palette, same naming rules (Rules A–J), same kind-folder layout, same ESLint configuration.

* REQUIREMENTS

The runtime stack is small — Node + npm cover the dev server, build, lint, tests, and the image conversion step that runs as =predev= / =prebuild=.

** Required (every contributor)
- *Node.js* =>= 20.0.0= (LTS) — enforced by the =engines.node= field in =package.json=. Verified on Node =22.x=.
- *npm* =>= 10.x= (bundled with Node 20+).
- *Git* =>= 2.30= for cloning and CI hooks.
- *POSIX shell* (bash / zsh) — required for =scripts/convert-fonts.sh=. On Windows, run via WSL2.

** Optional (only for specific scripts)
- *ImageMagick* =>= 7.0= — only for favicon generation (=npm run generate-favicons= / =npm run build-all=). Skip if you don't need favicons.
- *python3* =>= 3.9= with =fonttools= + =brotli= — only for the WOFF2 subsetting helper. Install with =pip install fonttools brotli=. Without it, =./scripts/convert-fonts.sh= falls back to whole-font =woff2_compress=.

** What ships with =npm install=
- =sharp ^0.34= — drives the WebP (q=90) + AVIF (q=75) variant generation in =scripts/convert-images.mjs= (wired as =predev= / =prebuild=).
- =vue ^3.5=, =vite ^6.0=, =vue-i18n ^9.14=, =@unhead/vue ^1.11= — runtime stack.
- =vitest ^2.1=, =@vue/test-utils=, =happy-dom=, =eslint ^9.17= + plugins — dev / test / lint stack.

* SETUP

#+BEGIN_SRC sh
git clone https://github.com/Kyonax/kyo-web-online.git
cd kyo-web-online
npm install
npm run dev   # dev server on http://localhost:9000
#+END_SRC

The first =npm install= fetches sharp's prebuilt binaries — no compiler toolchain needed on Linux / macOS / Windows-WSL2. The =predev= hook regenerates AVIF + WebP variants from any new JPG / PNG in =src/assets/app/= the first time you start the dev server.

* DEV / BUILD COMMANDS
| Command                   | What it does                                                     |
|---------------------------+------------------------------------------------------------------|
| =npm run dev=               | Vite dev server at http://localhost:9000 (HMR, source maps).     |
| =npm run build=             | Production build to =dist/=. Runs =npm run precheck= as =prebuild=.    |
| =npm run preview=           | Serves the built =dist/= locally — sanity-check production output. |
| =npm run lint=              | ESLint over =**/*.{js,mjs,vue}= (CCS standards + Vue ruleset).     |
| =npm run lint:fix=          | ESLint with =--fix=.                                               |
| =npm test=                  | Vitest single-run.                                               |
| =npm run test:watch=        | Vitest watch mode.                                               |
| =npm run test:coverage=     | Vitest with v8 coverage.                                         |
| =npm run generate-favicons= | Grunt favicon generation (requires ImageMagick).                 |
| =npm run build-all=         | Sequentially runs favicon generation + production build.         |

** Validation scripts
Six gates that run before every build via =prebuild= (and wired into =.github/workflows/ci.yml=).

| Command                          | What it does                                                                          |
|----------------------------------+---------------------------------------------------------------------------------------|
| =npm run check:i18n=             | Locale parity gate — every key in any locale exists in every locale; =RAW_HTML_KEYS= entries all resolve. |
| =npm run check:i18n-keys=        | Every =t('...')= / =<i18n-t keypath>= reference resolves to a real key.               |
| =npm run check:trans=            | Asserts ZERO =[trans=]= / =kyo:language-changed= / =class-scheduler= references in =src/=. |
| =npm run check:color=            | Per-SFC 60/30/10 color-tier audit + zero hardcoded color literals (scans =<style>= AND comments). |
| =npm run check:aliases=          | Vite alias map ↔ ESLint resolver kept in sync; every aliased folder exists.           |
| =npm run check:licenses=         | CCS license preamble on every committed =.js= / =.mjs= / =.vue= file.                 |
| =npm run precheck=               | Composite — runs all six validators with a PASS/FAIL summary. Wired as =prebuild=.   |

** Image / font conversion scripts
| Command                          | What it does                                                                          |
|----------------------------------+---------------------------------------------------------------------------------------|
| =npm run convert:images=         | Sharp-based WebP (q=90) + AVIF (q=75) generator over =src/assets/app/=. Wired as =predev= / =prebuild=. |
| =npm run convert:images:force=   | Same, but regenerates every variant (use after swapping a source image).             |
| =npm run convert:fonts=          | TTF → WOFF2 conversion via =woff2_compress=.                                          |
| =npm run convert:fonts:subset=   | TTF → WOFF2 with per-family unicode-range subsetting (requires =pyftsubset=).         |

** Feature flags (build-time)
Set the env var to override the defaults in =src/config/features.js=. Disabled features are tree-shaken from the build (zero bytes shipped).
| Variable                         | Default | Effect                                                                |
|----------------------------------+---------+-----------------------------------------------------------------------|
| =VITE_VIMEO_ENABLED=             | =true=  | Set =false= to omit the Vimeo embed entirely (no chunk, no preconnect, no facade). |
| =VITE_VIMEO_PRECONNECT=          | =true=  | Set =false= to keep the embed but skip the =<link rel="preconnect">= hint. |

#+BEGIN_SRC sh
# Production build with Vimeo disabled
VITE_VIMEO_ENABLED=false npm run build
#+END_SRC

* CONTRIBUTING
This project is licensed under [[file:LICENSE][GPL-2.0-only]].

Code style follows [[https://github.com/ccs-devhub][CCS]] standards, enforced via [[file:eslint.config.mjs][eslint.config.mjs]]. The linter mirrors the [[https://github.com/Kyonax/reckit/blob/master/eslint.config.mjs][reckit project's configuration]] verbatim with a Vue overlay layered on top.

Naming conventions and folder layout follow reckit's [[https://github.com/Kyonax/reckit][Rules A–J]] (recorded in =CODE_STANDARDS_MIGRATION.md= §3 and the =reckit_naming_conventions= org-roam node):
- =camelCase= functions
- =snake_case= variables
- =UPPER_SNAKE_CASE= constants
- =PascalCase= classes + component import bindings
- =kebab-case= filenames

Every committed =.js= / =.mjs= / =.vue= file carries the CCS license preamble (verified by the =check:licenses= gate). Every PR must pass =npm run precheck= (locale parity, key resolution, banned-reference check, color-tier audit, alias sync, license headers).

The CI pipeline at =.github/workflows/ci.yml= enforces the gates on every push and pull request to =main= / =develop= / =vue-migration=.

* DEPLOY
Two GitHub Actions workflows (=.github/workflows/=) deploy the built =dist/= to dedicated branches for static hosting:
- =deploy-to-build-main.yml= — pushes =main= → =build-main= branch (production at https://kyonax.github.io/kyo-web-online/).
- =deploy-to-build-dev.yml= — pushes =develop= → =build-dev= branch (staging).

Both run =npm run build-all= (favicons + production build) and publish via =s0/git-publish-subdir-action=.
