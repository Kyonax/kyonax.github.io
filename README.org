* WHAT IS KYO-WEB-ONLINE?
The personal portfolio site for *[[https://github.com/Kyonax][Cristian D. Moreno (Kyonax)]]* at [[https://kyonax.com]], a multilingual (EN / ES) cyberpunk-themed single-page web app showcasing experience, active projects, FAQ, and contact channels. Statically rendered per locale with =vite-ssg= so every route ships pre-baked HTML for SEO, AI-engine answer extraction, and instant first paint.

The codebase migrated from native Web Components + Webpack to *Vue 3 (=<script setup>= + Composition API) + Vite* in May 2026. Conventions mirror the sibling [[https://github.com/Kyonax/reckit][reckit]] project, same SCSS palette, same naming rules (Rules A–J), same kind-folder layout, same ESLint configuration.

* PREREQUISITES

** Required
- *Node.js* =>= 20.0.0= (LTS), enforced by the =engines.node= field in =package.json=. Verified on Node =22.x=.
- *npm* =>= 10.x= (bundled with Node 20+).
- *Git* =>= 2.30= for cloning and CI hooks.
- *POSIX shell* (bash / zsh), required for =scripts/convert-fonts.sh=. On Windows, run via WSL2.

** Optional (specific scripts only)
- *python3* =>= 3.9= with =fonttools= + =brotli=, only for the WOFF2 subsetting helper. Install with =pip install fonttools brotli=. Without it, =./scripts/convert-fonts.sh= falls back to whole-font =woff2_compress=.
- *pyfiglet*, only for generating new Tier 1 file-header place-name ASCII art. Install with =pip install --user pyfiglet=.

** Shipped with =npm install=
- =sharp ^0.34=, drives the WebP (q=90) + AVIF (q=75) variant generation in =scripts/convert-images.mjs= (wired as =predev= and =prebuild=).
- =vue ^3.5=, =vite ^6.0=, =vite-ssg=, =vue-router=, =vue-i18n ^9.14=, =@unhead/vue ^1.11= for the runtime stack.
- =vitest ^2.1=, =@vue/test-utils=, =happy-dom=, =eslint ^9.17= plus plugins for the dev, test, and lint stack.

* SETUP

#+BEGIN_SRC sh
git clone https://github.com/Kyonax/kyo-web-online.git
cd kyo-web-online
npm install
npm run dev   # dev server on http://localhost:9000
#+END_SRC

The first =npm install= fetches sharp's prebuilt binaries, no compiler toolchain needed on Linux, macOS, or Windows-WSL2. The =predev= hook regenerates ASCII art rasters, AVIF + WebP variants, and the sitemap before the dev server starts.

** Validation gates
Eight checks run before every build inside =precheck= (wired as =prebuild= and into =.github/workflows/ci.yml=). =check:seo= is separate and runs as =postbuild= against the built =dist/=.

| Command                          | What it does                                                                                       |
|----------------------------------+----------------------------------------------------------------------------------------------------|
| =npm run check:i18n=             | Locale parity, every key in any locale exists in every locale and every =RAW_HTML_KEYS= entry resolves. |
| =npm run check:i18n-keys=        | Every =t('...')= or =<i18n-t keypath>= reference resolves to a real key.                            |
| =npm run check:trans=            | Asserts ZERO =[trans=]=, =kyo:language-changed=, or =class-scheduler= references in =src/=.         |
| =npm run check:color=            | Per-SFC 60/30/10 color-tier audit plus zero hardcoded color literals (scans =<style>= AND comments). |
| =npm run check:aliases=          | Vite alias map and ESLint resolver kept in sync, every aliased folder exists.                       |
| =npm run check:licenses=         | CCS license preamble on every committed =.js=, =.mjs=, or =.vue= file.                              |
| =npm run check:json-ld=          | Validates the JSON-LD =@graph= shape plus required fields per entity type (Person, WebSite, FAQPage). |
| (precheck-only)                  | =scripts/check-projects-media.mjs= validates every =PROJECTS[*].images[]= entry resolves cleanly.    |
| =npm run precheck=               | Composite, runs every validator with a PASS/FAIL summary. Wired as =prebuild=.                      |
| =npm run check:seo=              | Crawls the built =dist/= and audits =<title>=, meta description, canonical, hreflang, JSON-LD. Wired as =postbuild=. |

* EDITING CONTENT

Almost every reader-facing string, project, image, and icon lives in =src/data/= or =src/assets/=. The validation gates (=precheck=) catch missing keys, broken references, and unconverted images before they reach =dist/=.

** Projects (now-shipping plus featured cards)
Edit =src/data/projects.js=. The =PROJECTS= map keys the cards on the landing, and each entry follows this schema.

#+BEGIN_SRC js
'project-slug': {
  name: 'PROJECT NAME',                  // uppercase, displayed verbatim
  description: 'ONE-LINE TAG',           // uppercase, sits under the name
  url: 'https://github.com/...',         // empty string hides the external link
  featured: true,                        // true → "FEATURED" grid, false → "NOW SHIPPING" carousel
  status: 'IN_PROGRESS',                 // see PROJECT_STATUS keys below
  version: 'v0.3.0',                     // SHIPPED version (v0.4.0-dev goes in the description body)
  modality: 'REMOTE',                    // OR version, never both (used for client engagements)
  started: 'Nov 03 08:00:00 2025',       // WORKING_ON cards show "STARTED IN" count-up
  deadlines: {                           // "Mon DD HH:MM:SS YYYY" in Bogotá local time
    'dev release v0.4.0': 'May 25 21:00:00 2026',
  },
  images: ['project-slug.jpg'],          // file basename in src/assets/projects/ (JPG required)
  stack: ['vue', 'vite', 'eslint'],      // brand-icon slugs (see Skills grid below)
}
#+END_SRC

- *Status values* are defined at the top of the same file (=PROJECT_STATUS=). The full set is =WORKING_ON=, =DONE=, =IN_PROGRESS=, =ON_HOLD=, =ON_TODO=, =LIVE=, =DEPRECATED=, =UPDATING=, =RELEASE=. Each maps to a color and an i18n label key.
- *Deadlines* drive the countdown card. The worker picks the earliest =utc_ts > now=. Past deadlines stay in the object as historical record (do not delete them).
- *Images*. Drop =project-slug.jpg= into =src/assets/projects/=, then run =npm run convert:images:force=. WebP plus AVIF variants generate automatically.
- *YouTube entries*. An =images[]= slot accepts a full YouTube URL (=https://youtu.be/<id>= or =https://www.youtube.com/watch?v=<id>=). The modal carousel renders it as a privacy-respecting facade (poster plus click-to-play). The =projects-media= precheck gate validates every URL.
- *Long-form descriptions* live in =src/data/snippets.js= under =kyo-web.content-data.projects.<slug>.description= (HTML allowed, see *Translation strings* below). The 4-paragraph flow convention (problem, purpose, tech, status) is documented in =docs/= and in the session memory.

** Translation strings (EN and ES)
=src/data/snippets.js= is the single source of truth. It exports =TRANSLATIONS= with mirrored =en= and =es= trees under =kyo-web.{contact,content-data,landing}=.

Three template patterns map to it.
1. =t('key.path')= for plain string interpolation.
2. =<i18n-t keypath="key.path" tag="..."><template #slot>...</template></i18n-t>= for named slots.
3. =v-html="t('key.path')"= for opaque HTML, allowed only for keys listed in =src/i18n/raw-html-keys.js=.

Hard rules (gates fail otherwise).
- Both locales must contain the same key set. =npm run check:i18n= validates parity.
- Every =t(...)= reference in templates and scripts must resolve. =npm run check:i18n-keys= validates references.
- *No em-dashes* in body copy (use commas, periods, or parentheses). Titles (=<title>=, =og:title=, =meta.title=) are the only exception.
- *No semicolons or colons* in body copy. The =:= inside URL protocols (=https://=) is fine.
- *@* characters in i18n strings encode as =&#64;=, because vue-i18n treats raw =@= as a linked-message marker. The =v-html= path decodes it for the DOM, and =stripHtml= decodes it for JSON-LD.
- General-audience framing. Descriptions address anyone curious about the work, never "recruiters" or "hiring managers".

** Experience timeline
Entries live in =src/data/snippets.js= under =kyo-web.content-data.experience.<slug>=. Each entry has =role=, =specs=, =description=, =bullets= (HTML =<li>= sequence), and =tools=. The bullets are copy-pasted verbatim from the CV PDF at =src/assets/cv/cv_cristian_d_moreno_<en|es>.pdf= and intentionally diverge between locales (do not reconcile them).

When swapping the CV PDFs, keep the filename pattern. The hero CTA links to them by hardcoded path.

** Skills grid
=src/data/data.js= holds the =SKILLS= object. Each category lists technology IDs that must match either a Nerd Font codepoint (rendered via =<span class="icon-glyph">=) or a brand SVG basename in =src/assets/brands/<id>.svg= (rendered via =<BrandIcon name="...">=).

The =BrandIcon= registry is glob-derived. Drop a new SVG in =src/assets/brands/= (=viewBox="0 0 24 24"=, =currentColor= fills, =kebab-case= filename matching the tech id) and it is auto-available everywhere. Pull paths verbatim from Simple Icons for fidelity.

** FAQ
=src/data/snippets.js= holds =kyo-web.landing.faq.items[]=. Each item has =question= plus =answer= (HTML allowed via =raw-html-keys.js= entries, so add the new key there). The FAQ section also emits a per-locale =FAQPage= JSON-LD block automatically.

** Author info, SEO defaults, social links
=src/data/data.js= holds =AUTHOR_INFO=, =SEO=, =SUPPORTED_LANGUAGES=, and =DEFAULT_LANGUAGE=. The =SEO.keywords= array feeds =<meta name="keywords">= and the JSON-LD =knowsAbout= node.

** Static assets
| Folder                         | What goes there                                                                            |
|--------------------------------+--------------------------------------------------------------------------------------------|
| =src/assets/app/=              | Hero portrait, landscape background, OG banner. Drop JPG or PNG and predev converts variants. |

| =src/assets/fonts/=            | TTF sources. =convert:fonts:subset= emits WOFF2 plus unicode-range subsets.                 |
| =public/=                      | Served as-is. Holds favicons, =og-banner.jpg=, =privacy/index.html= (en and es), =robots.txt=. |

** ASCII art
Sources live at =src/assets/ascii/<slug>.txt=. The alphabet is =░▒▓█= plus space only (foreign Unicode breaks the renderer).

Per-file directives override defaults via a =key value= line at the bottom of the source.

#+BEGIN_SRC
░░░░ … your ASCII art … ░░░░

left-alignment: -55
#+END_SRC

Supported directive, =left-alignment <px>= overrides the global =ASCII_CENTER_OFFSET_X=. Run =npm run convert:ascii:force= after edits and it regenerates JPG, WebP, and AVIF for every art at the right scale, centering, and font size.

* DEPLOY
Two GitHub Actions workflows push the built =dist/= to dedicated branches for static hosting.

- =deploy-to-build-main.yml=, =main= → =build-main= branch. Production at [[https://kyonax.com]], served by Hostinger.
- =deploy-to-build-dev.yml=, =develop= → =build-dev= branch (staging).

Both run =npm run build= and publish via =s0/git-publish-subdir-action=. Hostinger pulls =build-main= as the public web root. The =.htaccess= at the repo root handles trailing-slash normalization, locale routing, and cache-busting headers.
